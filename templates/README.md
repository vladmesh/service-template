# Каталог шаблонов сервисов

Шаблоны используются скриптом `add_service.sh` для создания новых микросервисов.

## Как устроено
- `services/default` — стек-независимый минимальный набор файлов (каталоги `src/`, `tests/`, общий Dockerfile, README, AGENTS).
- `services/python` — специализация для Python: включает пустой пакет `src/`, точку входа `src/app.py`, пример теста и Dockerfile на базе `python:3.12-slim`.

Каждый шаблон можно расширять собственными файлами, но минимальный набор должен сохраняться, чтобы автоматизация корректно работала.

## Создание нового шаблона
1. Добавьте каталог `templates/services/<type>`.
2. Вложите обязательные файлы (`Dockerfile`, `README.md`, `AGENTS.md`, директории `src/` и `tests/`).
3. Описывайте любые параметры, которые должен уметь подставлять `add_service.sh` (например, `__SERVICE_NAME__`).
4. Обновите документацию, чтобы команда знала о новом типе.
